# K2 · 切块的艺术：切在哪，决定查得准不准

[K1](../01-semantic-search/index.md) 学会了「按意思找」。但请注意：检索找的对象是**一段一段的资料**，而不是整篇文档。一份 50 页的员工手册，你没法整个塞进检索——得先把它切成一段段小块，每块单独入库。

切块（chunking）听上去像纯体力活：不就是把长文章剪短吗？可它偏偏是 RAG 里**最容易被忽视、又最容易翻车**的一步。切错了地方，前面 K1 的语义检索再强也白搭——因为你根本没把「完整的答案」切成一个能被找到的块。这一章讲清切块为什么重要、怎么会切坏，以及有哪些更聪明的切法。
