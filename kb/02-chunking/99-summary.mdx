# K2 · 小结与自测

## 一图回顾

```mermaid
flowchart LR
    D["📄 长文档"] --> C{"切块<br/>fixed / 按结构 / 语义 / late chunking"}
    C --> K1["块① + 元数据"]
    C --> K2["块② + 元数据"]
    C --> K3["块… + 元数据"]
    K1 & K2 & K3 --> E["编码入向量库"]
    C -.->|"原则"| P["好切块 = 每块一个完整的意思"]
```

一句话收束：切块是 RAG 里最不起眼、却最容易翻车的一步。**切太大**，一块塞太多事，重点被稀释、相似度被拉平；**切太小**，一个完整的事实被边界拦腰截断，只捞回半个答案。好切块的唯一标准，是让**每一块 = 一个完整的意思**——语义切块、滑窗、late chunking，都是在为这一个目标服务。
