# K6 · 评测：RAG 好不好怎么衡量

[K0](../00-why-rag/index.md) 到 [K5](../05-generation/index.md)，你已经能搭出一个会检索、会重排、答必有据的 RAG 了。但它到底**准不准**？

这是「能跑个 demo」和「能上生产」之间的分水岭。RAG 出错的方式五花八门——有时是检索没找对，有时是资料对了模型没照着用，有时是照着用了却复述走样。如果分不清是哪一环坏了，你只能瞎调一通。这一章给你一套体检工具：**RAG 三元组**（三盏诊断灯）和**三大失败模式**（三种病根），让你一眼看出该修检索还是修生成。
