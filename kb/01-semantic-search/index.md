# K1 · 语义检索：把「意思」变成坐标

[K0](../00-why-rag/index.md) 说 RAG 的第一步是**检索**——从知识库里找出相关资料。可这一步远没有听起来简单。

用户问「怎么退货」，你知识库里那段偏偏写的是「本店支持七天无理由退换」。两句话说的是同一件事，却**一个字都不一样**。老式的「按字面找」在这里直接歇菜。RAG 需要的是另一种检索：**按意思找**。

这一章拆开「按意思找」是怎么做到的——先讲清它的核心思想（把意思变成坐标），再讲支撑它跑在亿级数据上的引擎（向量数据库）。
