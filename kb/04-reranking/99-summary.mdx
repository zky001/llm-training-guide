# K4 · 小结与自测

## 一图回顾

```mermaid
flowchart LR
    KB[("📚 整个知识库")] --> R["① 召回 retrieve<br/>向量/混合检索<br/>快而粗 · 取 top-50"]
    R --> C["几十条候选<br/>（保证别漏）"]
    C --> RR["② 精排 rerank<br/>交叉编码器<br/>慢而准 · 读懂 query+doc"]
    RR --> T["top-3~5<br/>（保证排对）"]
    T --> G["🧩 交给生成"]
```

一句话收束：检索的黄金结构是**两阶段**——召回快而粗、保证「别漏」（召回率优先），精排慢而准、保证「排对」（精度优先）。精排的交叉编码器把查询和文档**拼在一起逐词读**，所以能分清「产假」和「陪产假」的一字之差；但正因为它必须成对现算、无法预存，慢到只配得起精排那几十条，配不起检索全库。
