# A0.2 自主性光谱：什么时候才需要真智能体

## 一个反直觉的行业共识

学完上一节，你可能已经跃跃欲试：把什么任务都交给行动循环！

但 2024 年底，做出 Claude 的 Anthropic 在其被业界引用最多的工程手册《Building Effective Agents》里，开篇说的却是一句泼冷水的话——大意是：**最成功的生产系统用的往往不是复杂的自主智能体，而是简单、可组合的模式；能不用智能体就不用。**

要理解这句话，得先把「自主性」摊开成一条光谱。

## 光谱：谁在决定下一步？

判断标准只有一条：**每一步「接下来干什么」是谁定的——人（写代码时）还是模型（运行时）？**

| 档位 | 下一步谁定 | 例子 | 特点 |
| --- | --- | --- | --- |
| ① 单次调用 | 人 | 翻译一段话、总结一篇文档 | 最便宜最稳 |
| ② <Term id="workflow">工作流</Term> | 人定步骤，模型填内容 | 「先提取要点 → 再写摘要 → 最后翻译」的固定三连 | 步骤可预测、可单测 |
| ③ 带工具的对话 | 模型可选择行动，但一轮为限 | 问天气时它自己去查一次 | 有限的自由 |
| ④ 智能体 | 模型定，循环开放 | 「把这个 bug 修好」——它自己决定读哪些文件、试几次 | 能力上限高，方差也高 |

②和④的分界最重要：**工作流的流程图是人画的，智能体的「流程图」是模型每步现画的。**

## 为什么「能简单就简单」

自主性每上一档，你都在用三样东西换灵活性：

- **可预测性**：工作流跑一千次是一千次相同的步骤；智能体跑两次可能是两条不同的<Term id="trace">轨迹</Term>（上一节深入层讲过：每步都是采样）。
- **成本与延迟**：④的 token 消耗常常是②的十倍以上——循环的每一圈都是一次完整的模型调用。
- **可调试性**：工作流哪步坏了一目了然；智能体失败时，你要在几十步轨迹里考古「它当时为什么这么想」（A7 章的核心难题）。

所以工程决策的顺序应该反过来：**先试①，不够再②，实在需要运行时的灵活决策才上④。** 用得上智能体的典型信号：任务的步骤数和路径**没法提前枚举**——比如修一个没见过的 bug、在陌生网站上完成订票。步骤能画成流程图的任务（发票报销审核、周报生成），画成流程图就好。

:::info 判断口诀
**流程画得出来 → 工作流；流程要看情况现编 → 智能体。**
:::

## 光谱在真实产品里的样子

拿几个你可能用过的东西对号入座（2025-2026 年的形态）：

- 输入法润色一句话：①单次调用；
- 会议纪要自动生成 + 发邮件：②工作流（步骤固定）；
- ChatGPT/Claude 联网搜索回答：③带工具的对话；
- Claude Code 修 bug、Deep Research 写调研报告：④智能体（步数开放，路径自定）。

值得注意的是，一个成熟产品往往是**光谱混搭**：外层是一个智能体循环，里面某些环节却是写死的工作流（比如「无论如何，提交代码前必须跑测试」）。把该锁死的锁死、该放开的放开，正是智能体工程的手艺所在——A4 章会给你五种「锁」的具体样式。

:::caution 这个划分在哪里不准确？
现实中光谱是连续的，档位是教学简化：比如「允许模型在 3 个预设分支里选一个」介于②和③之间；「智能体但每一步要人点确认」介于③和④之间。而且边界随模型能力漂移——2023 年必须写死的流程，2026 年的模型可能已经能可靠地自主处理。档位是用来做决策的语言，不是分类学真理。
:::

<DeepDive title="Anthropic 的原始分类与「围栏」设计">

《Building Effective Agents》（2024-12）的原始区分：

- **Workflows**：LLM 和工具通过**预定义代码路径**编排的系统（predefined code paths）；
- **Agents**：LLM **动态指挥自己的过程和工具使用**、掌控任务完成方式的系统。

文中给出五种工作流模式（提示链、路由、并行、编排者-工人、评估者-优化者，A4 章逐个拆解），并建议：从单次调用开始，只在「任务复杂度确实需要」时增加自主性；智能体适用于「步数不可预测、路径无法硬编码」的开放问题。

**自主性与围栏（guardrails）成对出现。** 放开自主性的同时收紧围栏是标准做法：

| 围栏 | 作用 |
| --- | --- |
| 最大步数 / 预算上限 | 防死循环与失控开销 |
| 工具白名单与权限分级 | 只读工具随便用，写操作要审批 |
| 高危动作人工确认 | 删除、支付、对外发送前停下来问人 |
| 沙箱执行 | 代码与文件操作圈在隔离环境里（A6/A8 章展开） |

OpenAI 的《A Practical Guide to Building Agents》（2025）给了一致的建议，并补充了单智能体 → 多智能体的演进原则：**在单个智能体 + 多工具走不通之前，不要引入多智能体**（A5 章再议）。

</DeepDive>

## 小结

:::tip 本节要点
- 自主性是条光谱：单次调用 → 工作流 → 带工具的对话 → 智能体；分界看「下一步谁定」。
- 自主性每上一档，都在用可预测性、成本、可调试性换灵活性——能简单就简单。
- 流程画得出来用工作流；路径要现场随机应变才用智能体；成熟系统常常混搭。
:::
