# A0.1 智能体的心跳：思考、行动、观察

## 一句话需求，两种命运

你对 AI 说：

> 「明天上午 10 点我约了客户在公司开会。看下明天北京的天气，要是下雨，把会改成线上并通知对方。」

**聊天机器人**的回答会很得体：「建议你查看天气预报，如果有雨，可以打开日历修改会议地点，并发消息通知客户……」——说得全对，但天气没查、会没改、通知没发。**它把你的任务原样还给了你。**

**智能体**收到同一句话，会真的去查、去改、去通知，最后告诉你一句「办完了」。亲眼看一遍它是怎么办的：

## 拆开看：里面根本没有新零件

把刚才的<Term id="trace">轨迹</Term>倒回去看，智能体只是在重复三拍：

1. **🤔 思考**：根据目前知道的一切，决定下一步干什么；
2. **🔧 行动**：调用一个工具（查天气、查日历、改日程）；
3. **👀 观察**：把工具的返回结果读进来，回到第 1 拍。

循环转到任务完成，输出最终回答。这个「思考 → 行动 → 观察」的循环就是<Term id="agent-loop">行动循环</Term>——智能体的心跳。学术圈把这种「想一句、做一步、看结果」的范式叫 <Term id="react">ReAct</Term>（2022 年提出，Reason + Act 的缩写）。

关键在于：**每一拍「思考」和「行动」，模型做的都是上篇讲了九章的那件事——预测下一个词。** 区别只有两处：

- 它生成的文本里可以出现一种特殊格式：**工具调用请求**（下一章细讲）；
- 生成不是一轮就结束：工具的结果会**拼回上下文**，让它接着生成。

:::info 一句话定义
**<Term id="agent">智能体</Term> = 大模型 + 工具 + 循环。** 模型负责决策，工具负责干活，循环负责把事办完。
:::

所以上篇的所有知识在这里全部续命：思考的质量取决于预训练和后训练（第 4、5 章）、每一步输出是采样出来的所以有随机性（3.6 节）、上下文窗口是它的工作记忆（8.2 节）。**智能体不是新物种，是大模型的「带手办事形态」。**

## 循环带来了什么，又带走了什么

多了循环，能力和风险一起翻倍：

| | 聊天（单轮生成） | 智能体（循环） |
| --- | --- | --- |
| 能做的事 | 生成文本 | 改变世界（真的会动你的日历） |
| 出错的代价 | 你看出来就行 | 错误的行动可能不可撤销 |
| 一次任务的成本 | 一轮生成 | 几轮到几百轮生成 + 工具执行 |
| 失败方式 | 说错话 | 每一步都可能错，错误还会**滚雪球**（A7 章细算这笔账） |

这张表就是下篇的课程大纲：怎么把「行动」做对（A1）、怎么把「思考」做稳（A2~A4）、怎么多个循环协作（A5~A6）、怎么衡量与兜底（A7~A8）。

:::caution 这个类比在哪里不准确？
「心跳」听起来像智能体有个天然的生命节律——其实没有。循环是一段普普通通的外层程序（一个 while 循环）在驱动，模型每一拍都是被动地收到上下文、吐出文本。「自主性」不在模型里，而在**我们允许这个循环转多久、给它什么权限**。这也是为什么 A8 章的安全问题最终是工程和制度问题。
:::

<DeepDive title="行动循环的伪代码与停机条件">

智能体框架千姿百态，内核几乎都是这十行：

```python
messages = [system_prompt, user_request]
while True:
    output = llm(messages, tools=tool_schemas)   # 一次「预测下一个词」
    if output.has_tool_call:                     # 模型选择了行动
        result = execute(output.tool_call)       # 外部程序真正执行
        messages += [output, tool_result(result)] # 观察结果拼回上下文
    else:                                        # 模型选择了直接回答
        return output.text                       # 循环结束
```

三个值得注意的工程细节：

**停机条件。** 上面的循环有两种自然出口：模型不再调用工具（认为办完了），或达到外部限制（最大步数、超时、预算上限）。生产系统必须有后者兜底——模型可能陷入「反复查同一个东西」的死循环。

**上下文单调增长。** 每转一圈，`messages` 就变长一截（工具结果有时非常大）。长任务里上下文管理是核心工程问题，A3 章专门处理。

**每一步都是采样。** 同一任务跑两遍，轨迹可能不同（3.6 节的老朋友）。这让智能体的测试和调试与传统软件截然不同——你不能只跑一遍就说它「能用」，A7 章的评测方法学由此而来。

**术语澄清**：本课程说的 agent 与强化学习教科书里的 agent（在环境中最大化奖励的抽象主体）同源但不同物——LLM 智能体是它在语言世界的具体实现，环境就是工具返回值构成的信息流。

</DeepDive>

## 小结

:::tip 本节要点
- 智能体 = 大模型 + 工具 + 循环；「思考 → 行动 → 观察」转到任务完成。
- 每一拍思考仍然只是「预测下一个词」——上篇的一切在这里继续有效。
- 循环让模型从「给建议」变成「办完事」，也让错误从「说错话」变成「做错事」。
:::
