# A5.2 协作形态：谁指挥谁

## 雇了团队，就得定组织架构

上一节把团队雇齐了，下一个问题立刻出现：**谁指挥谁？** 是设一个老板统筹全局的层级制，还是人人平等、吵出结果的圆桌会？

这个问题在人类组织学里吵了一百年，在多智能体系统里，2025-2026 年的生产实践已经给出了相当一边倒的答案。

## 形态①：编排者-工人——层级制，绝对主流

[A4 的五块积木](../04-orchestration/index.md)里有一块<Term id="orchestrator-worker">编排者-工人</Term>：中枢拆任务、派工人、收成品。把「工人」从单次调用升级成完整的<Term id="sub-agent">子智能体</Term>，就是多智能体世界的层级制——**当前生产系统几乎清一色是它**。

与其描述，不如看戏。下面是一台四人剧：编排者接到「学校机房要换 20 台训练用显卡，预算 15 万元」的采购调研任务（剧本为教学构造）：

步进看完这 14 幕，请回头盯三个地方：

1. **第 2~7 幕的并行阶段**：三名调研员同时开工（性能⚡、价格💰、风险🛡️），右侧各自的上下文条独立生长——墙钟时间只算最慢的那个；
2. **隔离的证据**：工人们 4~6k token 的搜索、比价、翻条款过程，全部留在自己肚子里；编排者只收到三条 500 token 的结论，它的上下文条全程停在 5k 左右——这就是上一节说的<Term id="context-isolation">上下文隔离</Term>在动态发生；
3. **第 11 幕的冲突仲裁**：性能首选（X4090-class）撞上预算红线——两个工人的结论打架了。注意是谁解决的：**编排者**追加了一轮定向调研（第 12~13 幕），然后拍板定稿。没有中枢的系统，这种冲突会变成无休止的拉扯。

## 形态②：平等协商与辩论——学术有效，生产少见

没有老板的圆桌会也存在：两个智能体就一个问题互相批驳若干轮，最后由裁判模型或多数投票收尾。学术研究（2023 年起的多智能体辩论工作）表明，这种「辩论」在部分事实与推理任务上确实能提高正确率——对抗视角逼出了单模型不会主动检查的漏洞。

但生产系统里它很少露面，原因很实际：**轮数难控**（吵到什么时候算完？）、**成本高**（每轮都是全员生成）、**收益不稳**（简单问题白吵，难问题可能一起错）。它更像研究工具箱里的手术刀，而不是流水线上的扳手。

## 形态③：评审流水线——固定接力

第三种形态介于两者之间：写手 → 审稿人 → 修订的**固定顺序接力**，即 [A4 评估者-优化者](../04-orchestration/01-five-patterns.mdx)的智能体版。没有动态调度（顺序写死），但有角色对抗（审稿人格专职挑刺）。代码评审、文稿质检这类「产出 + 把关」场景最常见——它保留了对抗的收益，又避开了圆桌会的失控。

## 人也是一个节点

还有一位常被漏算的「团队成员」：**你**。生产系统的标准做法（2025 年起几乎是标配）是把人设计成一个特殊的智能体节点——高危动作（删库、付款、对外发信）前，编排者向「人类节点」发一条消息、等待批准才继续。这就是 human-in-the-loop：它不是外挂的安全贴纸，而是消息协议里的一等公民。为什么必须有它，[A8 章](../08-safety-frontier/index.md)细讲。

:::caution 剧本之外的真实世界
剧场的剧本是教学构造，真实系统里没有一句台词是免费的：编排者的系统提示词、派活的任务书格式、工人的结论模板，全都要精心设计。行业共识是——**「派活派得清楚」是多智能体系统的第一生产力**：含糊的任务书 = 工人各自发挥 = 汇总灾难。深入层给出模板。
:::

<DeepDive title="编排者的工程细节">

**任务书模板。** 派给工人的消息至少四栏：

```text
目标：   调研候选显卡的市场价格与供货情况
边界：   只看国内渠道；不评价性能（那是别人的活）
产出格式：每型号一行：型号 | 单价 | 供货 | 信源
预算：   最多 10 次搜索 / 8k token
```

「边界」一栏最容易被省略、也最不该省——它防止工人越界重复别人的工作（token 白花），或几个工人对同一问题给出打架的答案（协调灾难）。

**结论模板。** 工人交回的消息也要有形：结论 / 依据（关键信源）/ 置信度 / 未尽事项。上一节说过通信即有损压缩——模板就是「指定哪些信息不许被压掉」。

**并发控制与失败重派。** 工人可能超时、跑偏、返回垃圾。编排者要像任务队列一样管理它们：设超时、验产出（结论是否符合模板、是否回答了目标）、不合格就带着反馈重派——这其实是 [A2.2 有依据的反思](../02-planning-reflection/02-reflection.mdx)在系统层的重演。

**冲突仲裁的三种策略。** 结论打架时按代价升级：①编排者直接裁决（有足够依据时）；②追加取证（像剧场第 12 幕那样定向补一轮调研）；③升级问人（依据不足且决策高危）。剧场里走的是②。

**辩论研究一句。** 多智能体辩论 + 多数投票在事实类问答上有可测的增益（2023-2024 年多项研究），但增益随任务难度与模型能力变化很大，且成本数倍于单模型——这解释了它为何停留在论文里多、进产线少。

</DeepDive>

## 小结

:::tip 本节要点
- 编排者-工人（层级制）是生产系统的绝对主流：并行、隔离、冲突有人裁。
- 辩论（圆桌会）学术上对部分任务有效，生产里因轮数难控、成本高而少见；评审流水线是保留对抗、避开失控的折中。
- 「派活派得清楚」是第一生产力：任务书带边界，结论带依据与置信度。
- 人是一个特殊节点：高危动作前的人工批准是消息协议的一部分。
:::
