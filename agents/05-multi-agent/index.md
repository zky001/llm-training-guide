# A5 · 多智能体系统

[A4 的「编排者-工人」](../04-orchestration/index.md)里，工人还只是单次模型调用：领任务、交产出，一轮结束。把工人升级成**完整的智能体**——自己带工具、自己转循环、自己管上下文——就得到了多智能体系统。

2025 年起，旗舰产品的深度调研、复杂编码功能背后几乎都是它：一个中枢拆任务，一群子智能体分头干活。但**多不等于好**——每多一个智能体，就多一份 token 账单、多一条会丢信息的传话链、多一个「意见不合谁说了算」的协调难题。

这一章讲清三件事：什么时候值得多、多了之后怎么协作、以及让「连接」变便宜的开放标准。
