# A4 · 小结与自测

## 一图回顾

```mermaid
flowchart TD
    Q1{"任务形状判断：<br/>步骤能提前定死吗？"}
    Q1 -->|"定得死 · 一条线"| C["⛓️ 提示链"]
    Q1 -->|"定得死 · 分几类"| R["🔀 路由"]
    Q1 -->|"定得死 · 可同时"| P["⚡ 并行<br/>（分片 / 投票）"]
    Q1 -->|"定不死 · 现场拆"| O["🎯 编排者-工人"]
    C --> Q2{"关键产出有及格线<br/>且迭代有效？"}
    R --> Q2
    P --> Q2
    O --> Q2
    Q2 -->|"有"| E["🔁 套一层评估者-优化者"]
    Q2 -->|"没有"| D["直接交付"]
```

一句话收束：**积木没有高低，只有合不合适**——步骤定得死选前三种，定不死用编排者，有及格线就加评估层；拼装时外层保守、里层灵活，拼到分支爆炸就把那一块放权给智能体。
