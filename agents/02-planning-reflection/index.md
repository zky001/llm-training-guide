# A2 · 规划与反思：把大事拆小，做完回头看

A1 给了模型一双手，但**手脚麻利不等于办事靠谱**。工具调用解决的是「能不能做」，这一章解决的是「做得对不对」——复杂任务需要两种「元能力」：

1. **动手之前把大事拆小**——规划：没有计划的智能体像没头苍蝇，转了二十圈还没碰到正题；
2. **动手之后回头检查**——反思：不检查就交付的智能体，会把错误一路带到终点。

「规划（planning）」和「反思（reflection）」是 2025-2026 年所有智能体框架的核心词汇，也是吴恩达总结的四大智能体设计模式中的两个（另外两个——工具使用和多智能体——分别在 A1 和 A5 章）。
