# A2.2 反思：自己当自己的审稿人

## 模型默认不回头

你写完一篇作文会回头读一遍，顺手改掉三个错别字。这个动作太自然了，以至于你可能没意识到：**模型默认不会这么做**。

它一路向前生成，写完就交——这是自回归的天性（上篇 [3.5 节](/docs/transformer/gpt-assembly)的循环：一次吐一个 token，每个 token 落笔无悔，没有「回退键」）。第 3 句里算错的数字，会被后面 50 句安然继承。

<Term id="reflection">反思</Term>做的事，就是把「回头读一遍」变成流程里的一步：

:::info 反思的基本形态
**生成 → 自查 → 修订 →（必要时再来一轮）**
:::

自查不是泛泛地「再看看」，而是对着清单挑毛病：事实对吗？约束满足了吗？代码跑得过测试吗？上一节 PlannerCompare 实验里策略③每步执行后的「预算检查」「时间检查」，就是把这份清单钉进了循环。

## 为什么有效：挑毛病比写对容易

反思乍看很可疑——同一个模型，写的时候不会，检查的时候就会了？

会的，而且有结构性原因：**生成和检查是两种不同的任务姿态**。生成是「顺着写」——在几万个可能的下一词里连续做几百次选择，错一步就偏航；检查是「挑毛病」——面对一份已经存在的答案，逐条对照标准做判断题。判断题比连续创作容易得多。

这个不对称你在上篇见过：[5.3 节](/docs/post-training/rlhf)里 RLHF 的全部杠杆就是「**判断比创作容易**」——人类写不出完美回答，但能一眼看出两个回答哪个好。反思是同一根杠杆，只是这次做判断的不是人，是模型自己。

## 关键区分：有依据的反思 vs 空想的反思

反思的效果天差地别，分界线在于**手里有没有新证据**：

- **有依据的反思**：拿着工具返回的硬事实回头看——测试的报错信息、搜索到的资料、执行的异常栈。模型对照证据能精确定位问题，改进立竿见影。写代码的智能体「跑测试 → 读报错 → 改代码」循环如此有效，就是因为每一轮反思都有报错信息当依据；
- **空想的反思**：没有任何新信息，只是把模型的答案还给它问「你确定吗？再想想」。提升有限，甚至可能**越改越错**——模型会在你的质疑压力下把对的改成错的（上篇 5.6 谄媚问题的另一种发作）。

这是 2023-2024 年多篇研究反复确认的共识，反思范式的代表作 Reflexion（2023）的核心设计也正是「拿**环境反馈**反思」，而不是闭眼冥想。一句话记住：**反思的质量取决于证据的质量。**

## 反思的账单

反思不是免费的：每轮自查和修订都是一次完整的生成成本。上一节的实验里，带反思的策略③花了约 4200 token，是一把梭（约 800）的 5 倍多——换来 3 条约束全中。什么任务值得付这笔钱，A0.2 的光谱思维依然适用：错不起的任务多反思，错得起的任务少反思。

:::caution 反思不是万灵药
三个边界要心里有数：**盲区自查不出**——模型不知道自己不知道的事，知识性错误往往反思十轮也发现不了（所以要靠外部证据）；**收益快速递减**——经验上反思超过 2-3 轮，改进近乎停滞，账单却线性上涨；**偏差是共享的**——「评估者」和「生成者」是同一个模型时，它们带着同一套偏见，自己批改自己的卷子总有盲点。请另一个模型当审稿人，正是 A5 章多智能体的动机之一。
:::

<DeepDive title="Reflexion 与 evaluator-optimizer">

**Reflexion（2023）**是反思范式的代表工作，结构是一个跨试次的循环：

执行任务 → 拿到环境反馈（测试通过率、任务得分）→ 把失败原因**语言化**（「我在第 3 步没检查边界条件」）存入记忆 → 带着这条反思重试。要点有二：反思的依据是环境的真实反馈，不是空想；反思以自然语言存进记忆（A3 章的话题），下一次尝试直接受益。在编码基准上，这种「带记忆的重试」显著优于盲目重试。

**evaluator-optimizer 模式**把生成者和评估者拆成两个角色循环（Anthropic 五种工作流模式之一，A4 章正式登场）：

```python
draft = generator(task)
for _ in range(max_rounds):
    review = evaluator(task, draft)      # 只挑毛病，输出通过/不通过+理由
    if review.passed:
        break
    draft = generator(task, feedback=review.reasons)
return draft
```

两个角色可以是同一个模型的两次调用（便宜，但偏差共享），也可以是不同模型（贵，但视角独立）。

**并行版反思：自我一致性（self-consistency，2022）。** 与其生成一份再反复修，不如并行采样多份、投票取多数（上篇 [8.4](/docs/frontier/reasoning-models) 提过）。它和串行反思正交，可叠加使用——并行保证起点不离谱，串行负责精修。

**研究结论一句话**：有外部反馈（测试、检索、执行结果）时，反思带来显著且稳定的提升；无外部反馈的纯自我修正效果不稳定，多篇 2023-2024 年的评估工作发现其在部分任务上为零甚至为负。

**过程奖励模型（PRM）**：更进一步，可以专门训练一个「挑错模型」按步骤给推理打分（2023 起用于数学推理），相当于把评估者从「提示词角色」升级成「专职裁判」——它也是上篇 8.4 深入层里过程奖励的近亲。

</DeepDive>

## 小结

:::tip 本节要点
- 模型默认一路向前不回头；反思把「生成 → 自查 → 修订」钉进流程。
- 有效的原因：挑毛病比写对容易——与 RLHF「判断比创作容易」同一根杠杆。
- 反思的质量取决于证据的质量：拿测试报错、搜索结果反思有效；空想的「你确定吗」可能越改越错。
- 反思按轮计费、收益递减，按任务风险决定投入几轮。
:::
