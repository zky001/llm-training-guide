# A7 · 评测与可靠性

前面六章教你把智能体造得越来越能干。这一章回答一个更冷酷的问题：**它到底靠不靠谱？**

这是分开「能跑个 demo」和「能上生产」的分水岭。智能体的评测比上篇讲的[模型评测](/docs/evaluation/benchmarks)难得多——任务是多步的、轨迹是随机的（同一输入两次结果可能不同）、失败方式千奇百怪。你没法像测传统软件那样「跑一遍绿了就发布」。

这一章先算清可靠性的**残酷算术**（为什么单步很准的智能体整体还是会翻车），再讲怎么**科学地衡量和排错**。吴恩达在他的 Agentic AI 课里反复强调一句话：评测驱动开发，是智能体工程真正的分水岭——凭手感调提示词的人，和用数据说话的人，做出来的东西不在一个可靠性量级上。
