# 8.3 多模态：把图像也切成 token

## 冰箱里有什么

拍一张冰箱内部的照片发给大模型，问「今晚能做什么菜」——它认出鸡蛋、西红柿和半根葱，然后建议你做西红柿炒蛋（这道菜是[5.3 节](../05-post-training/03-rlhf.mdx)的老朋友了）。

等等——前八章我们讲的模型只认识文字 token。**照片是怎么进 Transformer 的？**

## 万物皆可 token

答案朴素得出奇：**把图像也切成「词」。**

回想 [1.4 节](../01-language-models/04-embedding.mdx)：文字进模型前，先被切成 token、再变成向量——模型从头到尾只见向量，不见汉字。既然如此，图像凭什么不行？把一张图切成 16×16 像素的小方块（叫 **patch**），每个方块过一个编码器变成向量——一个方块，就是视觉世界的一个「词」。

$$
\text{一张图} \;\to\; \text{几百个小方块} \;\to\; \text{几百个向量} \;\to\; \text{和文字 token 排进同一个序列}
$$

这就是 ViT（Vision Transformer，2020）的核心洞察：图像块之于视觉，就像词之于语言。从此 Transformer 不挑食了——文字、图块、音频片段，只要能变成向量序列，注意力机制照单全收。

## 三段式拼接与「原生」路线

2023 年前后流行的<Term id="multimodal">多模态</Term>做法是**三段式拼接**（LLaVA 式）：一个现成的视觉编码器（把图变向量）+ 一个小小的投影层（把视觉向量「翻译」到语言模型的向量空间）+ 一个现成的 LLM。三段拼好后用图文对话数据微调——花小钱办大事，让开源社区迅速人手一个「能看图的模型」。

2024 年起，旗舰模型转向**原生多模态**（GPT-4o、Gemini 式）：不再拼接两个预训练好的模型，而是从头把图、文、音混在一起训练。最直观的收益是端到端语音对话——语音进、语音出，不再经过「语音转文字 → 思考 → 文字转语音」的三级接力，延迟从几秒降到几百毫秒，能听出你的语气、也能用叹气回应你。

顺带划清一条界线：以上讲的都是**理解**（图像进、文字出）。反方向的**生成**（文字进、图像出，如文生图）主力是另一条技术线——扩散模型，本书不展开，只需记住这是两套不同的机器。语音、视频同理：切片、变向量、进序列，配方一致。

:::caution 这个类比在哪里不准确？
「图像块 = 视觉的词」很好记，但有个重要差别：文字 token 来自一个离散的固定词表（[1.3 节](../01-language-models/03-tokenization.mdx)的 BPE），图像 patch 的向量是连续的、无穷多种。另外别高估「看得懂」：多模态幻觉比文字幻觉更隐蔽——图里没有的东西它也能言之凿凿地「看见」；数清图里有几只猫、判断左右方位、读长文档截图，都是 2025 年常见的翻车现场。
:::

<DeepDive title="从 ViT 到统一序列">

**patch embedding 的形状账**。一张 $224 \times 224$ 的图，按 $16 \times 16$ 切块：$(224/16)^2 = 14 \times 14 = 196$ 个 patch。每个 patch 有 $16 \times 16 \times 3 = 768$ 个像素值，过一个线性层变成 $d$ 维向量——所以一张标清图 ≈ 196 个「视觉 token」，再加位置编码（patch 也要发工牌），送进标准 Transformer。

**CLIP 与对比学习（2021）**。视觉编码器哪来？CLIP 用 4 亿「图-文」对训练：让配对的图和文字的向量互相靠近、不配对的互相远离（回想 [1.4 节](../01-language-models/04-embedding.mdx)——这是把「意思相近，向量相近」扩展到了跨模态）。训练出的视觉编码器天然「说着和语言模型相近的语言」，成为三段式拼接的标准零件。

**分辨率的代价**。高清图切出的 patch 数按分辨率平方增长：$1344 \times 1344$ 就是 7056 个 token——比一篇长文还长。视觉 token 直接计入上下文长度，吃掉 [6.2 节](../06-inference/02-kv-cache.mdx)算过的 KV Cache 显存。所以实际系统常用「低分辨率全图 + 高分辨率局部切片」的折中方案（2024 年起的常见工程配方）。

**交错图文数据**。原生多模态的训练语料是「图文交错」的网页序列（一段文字、一张图、又一段文字……），模型像读图文并茂的杂志一样做下一个 token 预测——[4.2 节](../04-pretraining/02-objective.mdx)的自监督配方原封不动，只是页面上多了图。

</DeepDive>

:::tip 本节要点
- 多模态的核心思想：万物皆可 token——图像切块变向量，与文字同序列处理。
- 三段式拼接（视觉编码器 + 投影层 + LLM）花小钱办大事；原生多模态从头混训，语音对话延迟大降（2024）。
- 理解（图进文出）与生成（文进图出/扩散模型）是两条技术线，别混为一谈。
- 多模态幻觉更隐蔽：数数、方位、长文档截图仍是常见短板（2025）。
:::
