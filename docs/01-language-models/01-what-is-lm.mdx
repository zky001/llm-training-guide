# 1.1 语言模型：预测下一个词的机器

## 你的手机里早就住着一个

在手机上打出「今天天气」，键盘上方立刻冒出三个候选：**「真好」「不错」「很冷」**。

先停下来想一秒：手机怎么知道该推荐这三个词，而不是「香蕉」「篮球」「微积分」？

答案是：输入法统计过海量的中文文本，发现「今天天气」后面**最常跟着**这几个词。它没有「理解」天气，它只是在做一件事——**根据你已经打出的字，猜下一个词最可能是什么**。

这就是<Term id="language-model">语言模型</Term>：

:::info 一句话定义
**语言模型 = 给定上文，输出「下一个词是什么」的<Term id="probability-distribution">概率分布</Term>的模型。**
:::

写成式子只有一个：

$$
P(\text{下一个词} \mid \text{上文})
$$

还记得 [0.3 节的「语言骰子」](../00-prerequisites/03-probability.mdx)吗？「今天天气真」后面接「好」的概率 50%、「不错」18%……语言模型的全部工作，就是**为任意上文造出这样一枚骰子**。生成整段话，则是掷骰子 → 把结果接到上文 → 再掷 → 再接……循环往复。

## 亲手玩一个真的语言模型

下面是一个**真的在你浏览器里运行**的迷你语言模型。它刚「读」完 40 句小学生日记水平的语料，学到的全部本事就是一张「上文 → 下一个字」的频率表。

建议按这个顺序玩：

1. 保持「上文长度：**1 个字**」，点几次「✍️ 自动续写 20 字」——它会写出**每两个字都通顺、连起来胡言乱语**的句子；
2. 点「↺ 重置」，切到「**2 个字**」再自动续写——连贯度肉眼可见地提升；
3. 多续写几轮，如果看到「⚠️ 数据稀疏」提示，别跳过：那正是老式语言模型的死穴（下一节细讲）；
4. 对比「🎲 按概率抽一个字」和「⭐ 总选概率最大的」——后者写出来的句子更「安全」但也更死板。

## 它和 GPT 干的是同一件事

请记住这个实验，因为**全书剩下的内容都建立在它上面**：

| | 这个迷你模型 | GPT / Claude / Qwen（2025 年的旗舰级） |
| --- | --- | --- |
| 任务 | 根据上文，给下一个字分配概率 | **完全相同** |
| 生成方式 | 掷骰子、接上、再掷 | **完全相同** |
| 「大脑」 | 一张查频率的表 | 数千亿<Term id="parameter">参数</Term>的神经网络 |
| 能看的上文 | 1~2 个字 | 几万到上百万个<Term id="token">词元</Term> |
| 读过的语料 | 40 句话 | 十几万亿词的互联网文本 |

所以，从第 2 章到第 4 章，整本书其实只在回答一个问题：**怎么把「查表」换成一个更聪明的东西**，让模型看得更远、猜得更准、还能举一反三。

:::caution 这个类比在哪里不准确？
「输入法联想」的候选来自词库规则加上简单统计，只看眼前几个字，也从不需要「懂」你在说什么。大模型虽然做同一个任务，但为了在几万字的上下文里把下一个词猜准，它被迫在训练中学会了语法、事实甚至推理（见下面的深入层）——「任务相同」不等于「能力相同」。另外，真实的大模型处理的不是「字」而是 token（1.3 节），预测范围是几万到二十几万个候选，而不是表里见过的那几个。
:::

<DeepDive title="自回归生成，以及它为什么蕴含理解">

**形式化。** 设词表为 $V$，一段文本是 token 序列 $x_1, x_2, \ldots, x_T$。语言模型学习条件分布：

$$
P_\theta(x_t \mid x_1, x_2, \ldots, x_{t-1})
$$

其中 $\theta$ 是模型参数。由概率的链式法则，整段文本的联合概率可以精确分解为逐词条件概率之积：

$$
P_\theta(x_1, \ldots, x_T) = \prod_{t=1}^{T} P_\theta(x_t \mid x_{<t})
$$

这就是**自回归**（autoregressive）分解：把「给整段话打分」这个大问题，无损地拆成 $T$ 个「预测下一个词」的小问题。生成时反过来用：从 $P_\theta(x_1)$ 采样得到 $x_1$，再从 $P_\theta(x_2 \mid x_1)$ 采样，如此迭代——这正是实验里「自动续写」按钮做的事，也是 ChatGPT 逐字往外蹦的原因。

**上面的迷你模型**是 $n$-gram：假设 $P(x_t \mid x_{<t}) \approx P(x_t \mid x_{t-n+1}, \ldots, x_{t-1})$，即只看最近 $n-1$ 个字，用语料频率直接估计。$n$ 稍大表就存不下、也数不准——1.2 节展开。

**为什么「预测下一个词」会逼出理解？** 想把这些填空题猜准：

- 「法国的首都是＿」——需要**事实**；
- 「她把书递给他，因为＿想读」——需要**指代消解**；
- 「358 + 267 = ＿」——需要**算术**；
- 侦探小说最后一页「凶手就是＿」——需要**通读全文的推理**。

在海量文本上把平均预测误差压到极低，最省参数的策略不是死记硬背（存不下），而是学会生成这些文本的**底层规律**。这就是「压缩即智能」的直觉：好的预测器必然是好的压缩器，而极致的压缩要求抓住规律本身。这个观点的严格版本可以从最小描述长度（MDL）原理展开，这里点到即止。

</DeepDive>

## 小结

:::tip 本节要点
- 语言模型只做一件事：**给定上文，输出下一个词的概率分布**；生成 = 反复「掷骰子 → 接上 → 再掷」。
- 你手机的输入法联想和 GPT 是同一任务的两个极端实现。
- 迷你模型靠查表，只能看 1~2 个字；后面所有章节都在回答「怎么把查表换成更聪明的东西」。
:::
