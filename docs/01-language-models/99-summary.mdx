# 第 1 章 · 小结与自测

## 全章鸟瞰：一条流水线

第 1 章讲的四件事，其实是同一条流水线上的四道工序。当你向大模型提问时，你的文字经历的是这样一趟旅程：先被**分词器**切成一串 token（1.3 节的积木），每个 token 换成编号；再拿编号去 **embedding 表**里查出各自的向量（1.4 节的「居住地址」）；这串向量进入**语言模型**，模型输出「下一个 token 是谁」的概率分布（1.1 节的语言骰子）；最后**采样**掷出一个 token，拼到句尾，整个过程周而复始——直到生成结束符。所谓「大模型在回答问题」，就是这个循环转了几百几千圈。

```mermaid
flowchart LR
    A["📝 文本<br/>「今天天气真」"] --> B["✂️ 分词<br/>(1.3)"]
    B --> C["🔢 token 序列<br/>[59, 60, ...]"]
    C --> D["📍 查 embedding<br/>(1.4)"]
    D --> E["➡️ 向量序列"]
    E --> F["🎲 语言模型<br/>(1.1)"]
    F --> G["📊 下一个 token 的<br/>概率分布"]
    G -->|采样| H["「好」"]
    H -->|拼回句尾，继续| B
```

第 2 章之前，唯一还是黑箱的就是中间那格「语言模型」——向量进去、概率出来，里面到底在算什么？这正是接下来两章的主线。

## 要点回顾

| 小节 | 核心要点 |
| --- | --- |
| [1.1 什么是语言模型](./01-what-is-lm.mdx) | 语言模型 = 给定上文，输出下一个词的概率分布；生成 = 反复掷「语言骰子」。<br/>连只会数数的 n-gram 小模型都能干这件事——它和 GPT 的差别在「看多远」和「怎么算」，不在目标。 |
| [1.2 从查表到神经网络](./02-history.mdx) | n-gram 靠死记硬背查表，上文一长就组合爆炸、处处「没见过」（数据稀疏）。<br/>神经网络语言模型靠向量与泛化打破僵局，才有了今天的大模型。 |
| [1.3 分词](./03-tokenization.mdx) | 模型只认编号；子词方案（BPE）在「太碎」与「词表爆炸」之间折中。<br/>BPE 训练 = 反复合并语料里最高频的相邻对；词表是训练语料的镜子。 |
| [1.4 词向量](./04-embedding.mdx) | Embedding 让「意思相近的词住得近」，规律得以跨词迁移。<br/>距离编码「像不像」，方向编码「关系」；坐标全靠训练自动调出来，没人排版。 |
