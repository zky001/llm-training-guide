# 1.3 分词：把文字切成模型认识的积木

## 模型的第一个难题：它不认识字

前面两节反复说，<Term id="language-model">语言模型</Term>在「预测下一个词」。但有个问题被我们悄悄跳过了：

**模型是个数学函数，它的输入输出只能是数字。**「你好」这两个字，它根本看不见——必须先把文字变成编号，比如「你 → 59 号，好 → 60 号」。

那么按什么单位编号？这个看似不起眼的选择，直接决定了模型的「视力」：

- **切得太碎**（英文按字母、中文按笔画）：`understanding` 变成 13 个字母。每个编号几乎不带意义，一句话变成超长序列，模型算起来又慢，又得费劲从碎渣里拼出意思。
- **按整词编号**：英文有几十万个词，而且新词永远造不完——「显眼包」「city 不 city」进不了旧词表，模型见到就抓瞎。这种词表里没有的词叫<Term id="oov">未登录词</Term>。更糟的是，`run`、`running`、`ran` 会被当成三个毫无关系的编号，明明是一家人却互不相认。
- **折中方案：子词（subword）**——常见的组合（「学习」「人工智能」）合成一块大积木，罕见的词拆成小积木去拼。<Term id="vocabulary">词表</Term>大小可控，又永远不会拼不出来。

主流大模型全部采用第三种。而造出这份「积木清单」的最经典算法，就是<Term id="bpe">字节对编码</Term>（BPE）。它简单得让人意外：

:::info 一句话定义
**BPE 的训练 = 在语料上反复做同一件事：数一数哪两个相邻积木最常一起出现，把它们粘成一块新积木。**
:::

**推荐玩法**：

1. 先点几次「🔗 合并一次」，按钮上会实时显示当前出现最多的那对组合——第一条规则几乎必然是「学」+「习」，因为这对搭档在语料里出现得最多；
2. 点「▶ 自动训练到 20 条」，盯着「语料 token 数」和「压缩率」一路下降，「深度学习」「人工智能」这样的四字积木会自己长出来；
3. 到下面的输入框切一切自己的句子。注意默认句子里的「我」和「和」带着 ❓——语料里从没出现过这两个字，它们就是这个小词表的未登录词。

## 训练一次，切用万次

玩完实验，BPE 的全貌其实就两句话：

- **训练阶段**（造积木清单，只做一次）：从单个字出发，重复「数频率 → 合并最高频的相邻对」，每合并一次就往词表里添一块新积木，直到词表达到预定大小。
- **使用阶段**（切句子，每次输入都做）：拿到新句子，先拆成单字，再把当年学到的合并规则**按学习时的先后顺序**重放一遍。语料里常见的组合自然被粘成大块。

所以「分词器切出来的结果」完全是训练语料的镜子：语料里「学习」常见，它就是一块积木；语料里没有「我」，「我」就只能当散件处理。

:::caution 这个实验在哪里不准确？
实验为了看得清，做了三处简化。① 真实分词器不在「汉字」层面工作，而在**字节**层面——任何语言、任何符号在计算机里都是字节，所以真实词表永远不会有 ❓（详见下方深入层）；② 规模差了好几个数量级：我们用一百字语料学 20 条规则，真实分词器用 TB 级语料学出几万到二十几万块积木；③ 一个真实的副作用值得知道：以 2024-2026 年主流分词器为例，训练语料里英文占大头，英文常见词往往独占一块积木，而不少汉字会被拆成 2~3 个字节碎片——**同样的内容，中文常常比英文更费 token**（也就更费钱、更快占满上下文窗口）。
:::

<DeepDive title="字节级 BPE 与词表大小的权衡">

**为什么是「字节对」编码。** 计算机中的文本都是 UTF-8 字节流：一个英文字母占 1 字节，一个常用汉字占 3 字节。字节只有 $2^8 = 256$ 种取值，byte-level BPE 就以这 256 个字节作为初始词表，在字节序列上做合并。好处是**彻底消灭未登录词**：任何字符——生僻字、emoji、乱码——都能拆回字节兜底表示，最坏情况只是切得碎一点。GPT-2（2019）首先采用这个方案，沿用至今。这也解释了正文里的中文现象：一个汉字 3 个字节，如果它没被合并进词表，就会被切成 2~3 个 token。

**词表大小是三方拉锯。** 设词表大小为 $V$、向量维度为 $d$：

- $V$ 越大 → 常见词都能独占积木，同样的文本切出的**序列越短**，推理越快、上下文利用率越高；
- 但 $V$ 越大 → 模型的 embedding 矩阵（$V \times d$，见下一节）和输出层参数越多，罕见 token 训练不充分；
- $V$ 越小 → 参数省了，但序列变长，注意力计算量随序列长度平方增长（第 3 章）。

实际数字（各模型技术报告口径）：GPT-2（2019）词表 50257；GPT-4o 的 o200k 词表约 20 万（2024）；Llama 3 约 12.8 万（2024）、Qwen 系列约 15 万（2024-2025）。词表整体在变大，一个重要动机就是对中文等非英语语言更友好。

**工程细节。** 真实分词器还要处理：预切分规则（不让合并跨越空格/标点，类似实验里不跨句号）、特殊 token（对话分隔符等）、数字切分策略。OpenAI 开源的 `tiktoken` 库可以直接查看这些规则。

</DeepDive>

## 小结

:::tip 本节要点
- 模型只认编号不认字；按什么单位编号是「太碎」与「词表爆炸 + 未登录词」之间的权衡，子词是折中答案。
- BPE 训练 = 反复合并语料里最高频的相邻对；使用 = 把学到的规则按顺序重放。
- 分词器是训练语料的镜子：常见组合成为大积木，没见过的内容只能拆碎兜底。
- 真实分词器在字节级工作、词表几万到二十几万；中文常比英文费 token（以 2024-2026 年主流分词器为例）。
:::
