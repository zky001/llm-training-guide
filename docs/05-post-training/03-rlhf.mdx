# 5.3 RLHF：从「哪个更好」中学习

## 一个朴素的观察

上一节留下的难题：怎么告诉模型「什么样的回答**更好**」？

让标注员为每个问题写出完美示范？太贵了——写一篇好回答动辄二十分钟，还要求标注员本身是写作高手；更尴尬的是，随着模型变强，人写的示范未必比模型自己写的好。

但换一个任务，事情突然变简单了：**给你两个回答，选出更好的那个**。几秒钟就能判断，不需要你会写，甚至不需要你说得清理由。

> 判断比创作容易——这就是 RLHF 的全部杠杆。

品酒师未必酿得出好酒，但一口就能喝出高下。<Term id="rlhf">RLHF</Term>（基于人类反馈的强化学习）要做的，就是把千万次这样的「一口高下」变成训练信号。

## 从偏好数据到更好的模型：三步流水线

**第一步：收集偏好数据。** 就是你刚做的事：同一个问题、两个（或多个）回答、标注员选出更好的。

**第二步：训练<Term id="reward-model">奖励模型</Term>。** 光有比较还不够——强化学习需要的是一个能给**任何**回答打分的裁判。于是用这几十万条比较，训练一个专门的模型：输入「问题 + 回答」，输出一个分数，要求它给人类偏好的那个回答打更高分。训练完成后，这位「电子品酒师」就学会了替人类的平均口味打分——而且不知疲倦、批量工作。

**第三步：强化学习。** 让语言模型不断生成回答，裁判逐一打分，然后朝着「分数更高」的方向调整参数。还记得 [0.4 节](../00-prerequisites/04-gradient.mdx)蒙眼下山找最低损失吗？同一套梯度功夫，这次是**蒙眼上山**：往奖励更高的方向爬。

## 人类的口味，本来就有噪声

如果你和「多数标注员」没有全部一致——恭喜，你体验到了 RLHF 最真实的一面：**人类偏好本身就有分歧**。有人喜欢简洁，有人喜欢详尽；同一个人上午和下午的标准都可能漂移。

真实项目靠两样东西压制噪声：几十页的**标注规范书**（什么算有用、什么必须拒答、遇到拿不准的怎么办），以及**多人交叉标注**同一条数据。即便如此，标注一致率也远达不到 100%。奖励模型学到的从来不是某个人的品味，而是几十万人口味的**统计平均**。

:::caution 这个实验在哪里不准确？
组件里的「多数标注员」是教学设定的参考答案，真实标注中不少对比的争议率相当高。更重要的是，这条流水线有个致命软肋没在实验里出现：裁判模型是**学出来的近似**，它有漏洞——模型会在强化学习中疯狂钻裁判的空子（灌水凑长度、堆砌自信语气），得了高分却没变好。这就是下一节要讲的 reward hacking，也是 RLHF 工程里真正的高墙。
:::

<DeepDive title="Bradley-Terry 与奖励模型">

**怎么把「比较」变成可训练的数学？** 经典工具是 Bradley-Terry 模型（1952，最早用于给棋手排名）：假设每个回答 $y$ 有一个潜在分数 $r(y)$，人类偏好 A 胜过 B 的概率是

$$
P(A \succ B) = \sigma\bigl(r(A) - r(B)\bigr)
$$

其中 $\sigma$ 是 sigmoid 函数（[2.1 节](../02-neural-networks/01-neuron.mdx)的老朋友）：分差越大，「A 赢」的概率越接近 1；分差为 0 时五五开。

**奖励模型的架构**朴素得出奇：拿一个预训练/SFT 过的 LLM，把「预测下一个词」的输出头换成「输出一个标量分数」的头。训练损失就是让人类选中的回答得分更高：

$$
L_{\text{RM}} = -\log \sigma\bigl(r_\theta(x, y_{\text{chosen}}) - r_\theta(x, y_{\text{rejected}})\bigr)
$$

注意：损失只关心**分差**，不关心绝对值——所以奖励模型的分数没有客观量纲，只是相对排序的量化。

**InstructGPT 三阶段（2022）**：SFT（1.3 万条示范）→ 奖励模型（3.3 万条比较）→ PPO 强化学习。这套配方直接催生了 ChatGPT，也定义了此后几年「后训练」的标准形态。

**RLAIF 与 Constitutional AI（Anthropic，2022）**：既然判断比创作容易，那能不能让**强模型自己当标注员**？Constitutional AI 给 AI 一部「宪法」（一组原则），让它依据原则比较、修订回答，用 AI 反馈替代大部分人类标注——这就是 RLAIF（RL from AI Feedback），如今与人类标注混合使用已是常态（2025 年）。

</DeepDive>

## 小结

:::tip 本节要点
- RLHF 的杠杆：判断比创作容易——用「哪个更好」的比较代替「写出完美示范」。
- 三步流水线：收集偏好数据 → 训练奖励模型（电子品酒师）→ 强化学习朝高分爬坡。
- 人类偏好有噪声、有偏差（偏爱长回答、吃谄媚），奖励模型学到的是统计平均口味。
:::
