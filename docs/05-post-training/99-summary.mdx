# 第 5 章 · 小结与自测

## 全章鸟瞰

```mermaid
flowchart LR
    A["基座模型<br/>（最强接话茬选手）"] -->|"5.2 SFT<br/>几万份示范"| B["会对话"]
    B -->|"5.3 偏好标注 → 奖励模型<br/>5.4/5.5 RL：PPO · DPO · GRPO"| C["懂好坏"]
    C -->|"5.5 RLVR<br/>可验证奖励"| D["会深想"]
    D --> E["Chat / 推理模型"]
    E -.->|"5.6 仍未解决"| F["幻觉 · 谄媚 · 越狱"]
```

一段话串起整章：第 4 章烧出来的**基座模型**只会「把最像的文本接下去」（5.1），于是先用几万份「问题 + 完美回答」给它上**示范课**（5.2 SFT，损失只算回答部分）；示范教得会格式、教不会分寸，所以请几十万人做「哪个回答更好」的**比较标注**，训练出会打分的**奖励模型**（5.3——判断比创作容易，这是 RLHF 的全部杠杆）；拿着裁判做强化学习时，必须用 **KL 橡皮筋**拴住模型防止它钻裁判的空子（5.4 PPO），嫌贵可以走捷径——**DPO** 直接从偏好学、**GRPO** 用组内比较省掉价值模型（5.5）；而数学和代码的「答案可自动验证」催生了 **RLVR**：奖励只认对错，模型自发学会多想一会儿，这就是推理模型（5.5）。流水线尽头，**幻觉、谄媚、拒答边界、越狱**四个问题依然无解——它们划出了对齐的边界，也划出了正确使用大模型的边界（5.6）。

## 要点回顾

| 小节 | 两行要点 |
| --- | --- |
| [5.1 基座模型](./01-base-vs-chat.mdx) | base 模型是接话茬选手：问它问题，它可能续写出十个问题。不是不知道，是没人教过它「要回答」。 |
| [5.2 SFT](./02-sft.mdx) | 用示范数据教对话格式；损失只算 assistant 部分（loss mask）。示范教格式，教不了分寸。 |
| [5.3 RLHF](./03-rlhf.mdx) | 「选哪个更好」比「写出完美回答」容易一万倍；几十万条比较 → 奖励模型 → 往高分方向调。 |
| [5.4 PPO](./04-ppo.mdx) | 裁判有漏洞就会被钻（reward hacking）；KL 惩罚拴住模型，PPO 保证每步别迈太大——代价是四模型马戏团。 |
| [5.5 DPO / GRPO / RLVR](./05-dpo-grpo.mdx) | DPO 一步监督学习替代两步；GRPO 组内比较省价值模型；RLVR 用对错当奖励，推理能力自己长出来。 |
| [5.6 对齐的边界](./06-limits.mdx) | 幻觉治不断根、谄媚源自标注偏差、拒答边界是价值判断、越狱说明知识还在——了解边界才会正确使用。 |
