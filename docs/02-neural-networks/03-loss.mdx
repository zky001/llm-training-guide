# 2.3 损失函数：给「错」定价

## 错和错不一样

两个学生做同一道题，正确答案是 70：一个答 69，一个答 20。都错了，但显然**错得不一样重**。如果老师给他俩记同样的过错，谁也不知道该往哪改、改多少。

模型也一样。[0.1 节](../00-prerequisites/01-what-is-ml.mdx)说过，训练 = 自动调参数让「偏差」变小——前提是先把「偏差」定义成一个**数**。这个给错定价的函数，就是<Term id="loss">损失函数</Term>。它是训练的指挥棒：定价标准长什么样，模型就往什么方向学。

## 回归的定价：MSE

预测分数、房价这类「输出一个数」的问题，最常用**均方误差**（MSE）：把每条数据的偏差平方后取平均——0.1 节实验里那些红色误差线，量的就是它。

为什么要**平方**，而不是直接取偏差？两个理由：

1. **大错重罚**：差 10 分的惩罚是差 1 分的 100 倍，模型会优先消灭离谱的错误；
2. **处处能算坡度**：平方函数光滑可导，[0.4 节](../00-prerequisites/04-gradient.mdx)的「下山」在任何位置都知道往哪走（绝对值在零点有个尖角，坡度没定义）。

## 语言模型的定价：交叉熵

轮到语言模型，MSE 就不合适了——预测下一个 <Term id="token">token</Term> 不是「输出一个数」，而是**几万选一的选择题**：模型交出的答卷是一张概率分布表（[1.1 节](../01-language-models/01-what-is-lm.mdx)），比如「下一个字是『好』的概率 50%、『冷』12%……」。

给这种答卷定价，业界标准是<Term id="cross-entropy">交叉熵</Term>，规则简单得出奇：

> **只看你给正确答案分配了多少概率，取负对数作为罚分。**

$$
\text{损失} = -\log\bigl(\text{模型给正确答案的概率}\bigr)
$$

感受一下这个定价表的脾气：

| 给正确答案的概率 | 罚分 | 点评 |
| --- | --- | --- |
| 1.0 | 0 | 满分，零惩罚 |
| 0.9 | 0.11 | 挺自信，轻罚 |
| 0.5 | 0.69 | 五五开，中罚 |
| 0.1 | 2.3 | 没把握，重罚 |
| 0.01 | 4.6 | 基本没猜到，罚分爆炸 |

概率每低一个数量级，罚分不是线性增加，而是**猛涨一截**——模型「错得越自信」，挨得越狠。

现在可以把「训练大模型」翻译成一句大白话了：

:::info 一句话点题
**预训练 GPT = 在十几万亿个词上，把「给正确的下一个词打的概率」的平均罚分压到最低。**每个词都是一道选择题，交叉熵是唯一的判分标准——就这么朴素。
:::

:::caution 为什么不拿 MSE 凑合用？
直觉版答案：配上概率输出时，MSE 有个致命脾气——模型**错得最离谱的时候，坡度反而最平**（罚分封顶、梯度趋零），模型在最该猛醒的地方躺平了。交叉熵相反：越离谱，罚分和坡度都越陡，一鞭子抽在最疼的地方。严格的梯度对比见下面深入层。
:::

<DeepDive title="交叉熵、KL 散度与困惑度">

**完整定义。** 目标分布 $p$ 与模型分布 $q$ 之间的交叉熵为

$$
H(p, q) = -\sum_{x} p(x) \log q(x)
$$

语言模型训练中，目标是 one-hot 分布（正确 token 概率为 1，其余为 0），于是求和只剩一项，退化为正文的形式：$H = -\log q(y_{\text{正确}})$。

**与 KL 散度的关系。** $H(p,q) = H(p) + D_{\mathrm{KL}}(p \,\|\, q)$，其中 $H(p)$ 与模型无关。所以最小化交叉熵等价于最小化模型分布到目标分布的 KL 散度——「让模型的概率表长得尽量像真实世界的概率表」。

**与困惑度的关系。** [1.2 节](../01-language-models/02-history.mdx)深入层提过的<Term id="perplexity">困惑度</Term>就是平均交叉熵的指数：$\mathrm{PPL} = \exp(\bar{H})$。交叉熵降 0.1，困惑度约降 10%——两个指标是同一件事的两种读数。

**为什么配 softmax 时梯度极其干净。** 输出层用 <Term id="softmax">softmax</Term> 把打分变成概率 $q$ 时，交叉熵对打分（logit）的梯度恰好是 $q - p$：模型概率减去目标概率，逐项相减，不多不少（推导略，一行链式法则）。错得越多，$q$ 与 $p$ 差得越大，梯度越大——这正是「一鞭子抽在最疼的地方」的数学形态。而 MSE 配 softmax 的梯度里会多乘一个 $q(1-q)$ 因子，在 $q \to 0$（错得离谱）时梯度反而消失。

**label smoothing 一句话。** 实践中有时把 one-hot 目标改成「正确答案 0.9、其余均分 0.1」，防止模型过度自信，这个小技巧叫标签平滑。

</DeepDive>

## 小结

:::tip 本节要点
- 损失函数是训练的指挥棒：先把「错」定价成一个数，模型才知道往哪改。
- 回归用 MSE：平方让大错重罚、处处可导。
- 语言模型用交叉熵：只看给正确答案的概率，取负对数——越自信的错，罚得越狠。
- 训练大模型 = 在海量文本上把这个平均罚分压到最低。
:::
