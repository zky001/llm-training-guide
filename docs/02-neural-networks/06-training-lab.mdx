# 2.6 训练实验室：亲眼看损失掉下来

## 万事俱备，点火

盘点一下这一章攒下的家当：

- **模型**（2.1、2.2）：神经元拼成的、想多弯就多弯的函数；
- **损失**（2.3）：给「错得多离谱」定价的评分标准；
- **引擎**（2.4、2.5）：梯度下降负责调参，反向传播负责供应梯度。

三件套合体，就是所有<Term id="neural-network">神经网络</Term>训练的完整流程——一个四步循环：

**前向算预测 → 损失打分 → 反向传播算梯度 → 更新参数**，然后回到第一步，循环成千上万次。

下面这个实验室里跑的就是这个循环，一个字都不多：一个 25 个参数的小网络，学 40 个带噪声的数据点。它每秒能循环几千次——GPT 的训练循环与它**完全同构**，只是每一步都放大了十亿倍。

请按顺序做四个实验：

**实验 ①：看一次完整的训练。** 保持默认（8 个神经元、η = 0.1），点「▶ 开始训练」。眼睛别盯着上图的橙色曲线，盯**下图的损失曲线**：先是一段陡降（模型快速抓住大形状），然后越来越平缓（在细节上精雕细琢）。这条曲线是每个炼丹师的心电图——真实训练里工程师整天整夜盯的就是它。

**实验 ②：把它训炸。** 「↺ 重置」，把 η 拉到 0.5 以上再训。多数情况下损失不降反升、一路飙到 💥 爆炸提示；就算侥幸没炸，损失曲线也是剧烈的锯齿。0.4 节的小球飞出山谷、2.4 节的沟壑发散，现在你在一次「真训练」里再次见到了同一件事。

**实验 ③：容量不够，神仙难救。** 重置后选「2 个神经元」（只剩 7 个参数），训上 1000 步。损失卡在半空再也下不去，橙色曲线怎么都贴不住波浪——两个折角画不出三道弯。这不是训练方法的错，是模型<Term id="hidden-layer">隐藏层</Term>太小、容量不足：**欠拟合**（2.2 节的积木不够用了）。

**实验 ④：容量太大，学过头。** 换「32 个神经元」（97 个参数，比数据点还多两倍）继续训。损失能压得非常低，但注意橙色曲线开始追着**每一个噪声点**拐弯——它在背题，不是在学规律。组件会在损失逼近噪声水平时提醒你「见好就收」：再训下去，降的是训练损失，坏的是真本事。这叫<Term id="overfitting">过拟合</Term>。

## 心电图的读法

损失曲线为什么前陡后缓？开局参数是随机的，随便一步都是大坡，损失自然掉得快；越接近谷底坡度越平（0.4 节：谷底梯度趋近零），每步的收益递减。所以有经验的炼丹师看曲线**形状**就能诊断：一路平——学习率太小或模型太小；锯齿狂跳——学习率太大；陡降后稳稳走平——一切正常。

:::caution 这个玩具和真实大模型训练差在哪？
量级和细节都差得远，值得逐条说清：这里 40 个数据点全批量训练，真实训练是十几万亿 token 里随机抽 mini-batch（2.4 节）；这里几秒收敛，GPT 级预训练要几千张 GPU 跑数月，中途还要定期存<Term id="checkpoint">检查点</Term>防崩溃（第 4 章细讲）；这里只盯一条训练损失，真实训练还要同时盯**验证集**损失来抓过拟合（见深入层）；这里的学习率恒定，真实训练有 warmup 和衰减日程。但四步循环本身，一字不差。
:::

<DeepDive title="训练循环的代码形态与超参数">

**PyTorch 风格的训练循环。** 这一章的全部内容，落到代码里就十行：

```python
model = MLP(hidden=8)                       # 2.1-2.2：模型
loss_fn = nn.MSELoss()                      # 2.3：损失
opt = torch.optim.AdamW(model.parameters(), lr=1e-3)   # 2.4：优化器

for step in range(num_steps):
    x, y = sample_batch(data)               # 2.4：抽一小批
    y_hat = model(x)                        # 前向
    loss = loss_fn(y_hat, y)                # 打分
    loss.backward()                         # 2.5：反向传播，梯度就位
    opt.step()                              # 更新参数
    opt.zero_grad()                         # 清空梯度，准备下一轮
```

训练 GPT 的核心循环长得几乎一样，只是 model 换成 Transformer、loss 换成交叉熵、外面包上分布式并行（第 4 章）。

**常见超参数速览。** 超参数 = 不由训练自动调、需要人（或调参实验）事先定的旋钮：

| 超参数 | 人话 | 定错了会怎样 |
| --- | --- | --- |
| 学习率 | 每步迈多大 | 大→爆炸；小→龟速 |
| batch 大小 | 每步看多少数据 | 小→噪声大；大→费显存、泛化可能变差 |
| 训练步数 | 循环多少轮 | 少→欠拟合；多→过拟合+烧钱 |
| 隐藏层宽度/层数 | 模型容量 | 小→学不会；大→易背题、算力翻倍 |
| 参数初始化 | 从山上哪里出发 | 差的初始化让深网络一开始就梯度消失/爆炸 |

**验证集与早停。** 把一部分数据藏起来不参与训练（验证集），定期用它测损失。训练损失一直降、验证损失开始回升，就是过拟合的铁证——在回升点停止训练（早停）是最朴素有效的对策。大模型时代的说法叫「监控 held-out loss」，本质相同。

**对数坐标的意义。** 损失常在几个数量级间变化（4.0 → 0.04），线性坐标下后期变化被压成一条平线什么也看不出；取对数后，「每下降一格 = 缩小十倍」，前后期的进展都清晰可读。第 4 章的 scaling law 曲线也全部生活在对数坐标里，现在就习惯它。

</DeepDive>

## 小结

:::tip 本节要点
- 训练 = 四步循环：前向 → 损失 → 反向传播 → 更新，从 25 参数的玩具到千亿参数的 GPT 完全同构。
- 损失曲线是训练的心电图：陡降后平缓为正常，锯齿为学习率过大，平躺为容量或学习率不足。
- 容量太小学不会（欠拟合），容量太大背噪声（过拟合）；靠验证集损失判断何时「见好就收」。
- 真实大模型训练的差异在规模与工程（mini-batch、数月、checkpoint、验证集监控），不在原理。
:::
