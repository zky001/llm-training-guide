# 6.5 蒸馏与小模型：把大象装进口袋

## 不能什么都靠万卡机房

手机上的实时翻译、汽车里的语音助手、断网也要能用的笔记摘要——这些场景不可能每次都连回云端的万卡集群：延迟等不起、流量费不起、隐私也未必允许。**端侧需要小而强的模型**。

前两节的量化是「给大模型减肥」；这一节换个思路：干脆培养一个**小号继承人**。

## 蒸馏：不只抄答案，还要看解题思路

<Term id="distillation">蒸馏</Term>的设定很像师徒制：大模型当**老师**，小模型当**学生**，学生的目标不是从原始数据里从头苦学，而是**模仿老师**。

关键的巧思在于模仿什么。假设老师看一张图，它的判断是：

> 猫 70%、狗 25%、老虎 4%、汽车 0.01%……

如果只把「答案是猫」教给学生（硬标签），信息就只有一个字。但老师的**完整概率分布**（软标签）藏着多得多的知识：狗和猫很像、老虎有点像、汽车完全不像——这些「错误选项之间的相对关系」正是老师多年功力的浓缩。学生同时学习整张概率表，等于把老师的「审题直觉」一并继承了。

语言模型完全同理：老师对「下一个词」输出的那张几万项的概率表（你在 [3.6 节](../03-transformer/06-sampling.mdx)的采样实验里亲手拨弄过的那种分布），就是学生的教材。

## 数据蒸馏：老师直接写教科书

到了大模型时代，最流行的蒸馏形态其实更简单粗暴：**让老师大量生成优质文本，学生把它们当普通训练数据学**（序列级蒸馏 / 数据蒸馏）。

最出名的例子：DeepSeek-R1（2025 年）发布时，官方同步放出了一批「R1-Distill」小模型——用 R1 生成的**长推理轨迹**（完整的思考过程 + 答案）去微调 Qwen、Llama 系列的小模型。结果是 7B、14B 级别的小模型也学会了「先想后答」的做题风格，数学能力大涨——老师把「会思考」这件事写成了教科书，学生照着念也能学到几分。

## 「够用就好」的小模型哲学

小模型当然有天花板：知识容量、复杂推理都比不过大模型。但 2024-2026 年的端侧潮流证明了另一件事：**很多场景根本不需要天花板**。手机上的翻译、摘要、语音指令，3B 级别的模型已经够用；真遇到难题，再转发云端大模型兜底——「小模型守门、大模型压阵」的分工已是行业常态。

最后把三个「变小」的手段放在一起对比：

| 手段 | 改的是什么 | 一句话 |
| --- | --- | --- |
| 量化（[6.3 节](./03-quantization.mdx)） | 数字精度 | 同一个脑子，记数记粗一点 |
| 蒸馏（本节） | 知识载体 | 把知识教给一个新的小脑子 |
| 剪枝 | 网络结构 | 剪掉不重要的神经连接 |

三者可以叠加：先蒸馏出小模型，再量化到 int4——口袋里的大象就是这么装进去的。

:::caution 「蒸馏」这个词已经被用宽了
如今业界把「用一个模型的输出去训练另一个模型」统统叫蒸馏，与 Hinton 2015 年提出的「软标签蒸馏」原始定义已有不小距离——读文章时留意上下文。另外，用别家闭源模型的输出来训练自己的模型，可能违反对方的服务条款，这是工业界一个现实的合规灰区，本书点到为止。
:::

<DeepDive title="蒸馏的损失函数">

**软标签蒸馏（Hinton 2015）。** 学生同时学两个目标：贴近真实答案（硬标签交叉熵），以及贴近老师的输出分布（KL 散度）：

$$
L = \alpha \cdot L_{\text{CE}}(y, p_s) + (1-\alpha) \cdot T^2 \cdot \mathrm{KL}\bigl(p_t^{(T)} \,\|\, p_s^{(T)}\bigr)
$$

其中 $p_t^{(T)}, p_s^{(T)}$ 是老师和学生在**温度 $T$** 下软化后的分布（logits 除以 $T$ 再 softmax，正是 [3.6 节](../03-transformer/06-sampling.mdx)那个温度）。$T > 1$ 把尖锐的分布抹平，让「老虎 4%、汽车 0.01%」这类小概率的相对关系变得清晰可学——Hinton 称之为**暗知识**（dark knowledge）。系数 $T^2$ 用于平衡两项梯度的量级。

**序列级蒸馏。** 让老师生成完整文本 $\hat{y} \sim p_t$，学生对它做普通的下一词预测训练——形式上就是一次 [SFT](../05-post-training/02-sft.mdx)，只是「示范」由老师而非人类书写。它只传递了老师的「最优路径」而丢掉了整张概率表，理论上信息更少，但工程上极其简单（只需要老师的 API 输出，不需要 logits），因此成为大模型时代的主流形态。

**三个里程碑。**
- **Hinton et al. 2015**：提出软标签蒸馏与暗知识概念，奠定框架。
- **DistilBERT 2019**：把 BERT 压小 40%、保留约 97% 能力，证明蒸馏在预训练语言模型上可行。
- **R1-Distill 2025**：用推理模型的思维链轨迹做序列级蒸馏，让小模型继承「先想后答」的行为模式——蒸馏的对象从「知识」扩展到了「思考方式」。

</DeepDive>

## 小结

:::tip 本节要点
- 蒸馏 = 学生模仿老师；软标签（完整概率分布）比硬标签（只有答案）多出「错误选项相对关系」这层暗知识。
- 当代最流行的形态是数据蒸馏：老师生成优质文本（甚至推理轨迹），学生当 SFT 数据学。
- 量化改精度、蒸馏换脑子、剪枝改结构——三者可叠加。
- 端侧哲学：很多场景不需要最强模型，「小模型守门、大模型压阵」。
:::
