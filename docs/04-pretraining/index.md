# 第 4 章 · 预训练：真正的「炼丹」

前三章画好了全部图纸：模型架构（Transformer）、评分标准（交叉熵损失）、训练算法（梯度下降 + 反向传播）。图纸在手，接下来是**开进工厂**——

真正训练一个大模型时，工程师每天操心的不是公式，而是四件非常「工业」的事：

1. **原料**——十几万亿 token 的文本从哪来？怎么把互联网这个大垃圾场淘洗成干净语料？（4.1、4.2）
2. **预算**——参数开多大、数据喂多少、要烧几个亿？Scaling Laws 让这笔账可以提前算。（4.3）
3. **产线**——模型大到一张 GPU 根本放不下，几千张卡怎么分工协作？（4.4）
4. **事故**——训练跑到第 37 天突然崩了怎么办？（4.5）

这部分是市面上最缺系统性中文资料的内容，也是「大模型工程师」和「会调 API 的人」之间真正的分水岭。
