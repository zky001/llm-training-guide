# 4.1 数据从哪来：喂饱一个大模型

## 先算一笔阅读账

旗舰级大模型的<Term id="pretraining">预训练</Term>语料是**十几万亿个 <Term id="token">token</Term>**（2025 年的量级）。这是什么概念？

一个人每天读 8 小时书，按每分钟 300 字算，一天大约能读十万 token。照这个速度不吃不睡地读，读完十几万亿 token 需要**几十万年**——从直立人时代读到今天都读不完。

世界上没有任何一家出版社能供应这么多文字。唯一的来源，是**互联网本身**。

## 语料的配方

真实的预训练语料是多种来源按比例调配出来的，每种原料补一种「营养」：

| 来源 | 典型代表 | 贡献什么能力 |
| --- | --- | --- |
| 网页抓取 | Common Crawl（公益组织持续抓取全网的快照） | 占大头：语言的广度、常识、时效话题 |
| 代码 | GitHub 等开源仓库 | 严密的逻辑和结构化思维——代码读得多的模型推理更强 |
| 书籍 | 电子书库 | 长篇连贯性、叙事能力、深度知识 |
| 论文 | arXiv 等 | 科学知识、严谨表达 |
| 百科 | 维基百科等 | 高密度、相对可靠的事实 |
| 问答与对话 | 论坛、问答站 | 「一问一答」的语用习惯 |

各来源按什么比例混合（**数据配比**），直接影响模型的「性格」和强项——这是每家公司最核心的机密之一，论文里往往只有一句含糊带过。

## 互联网是个垃圾场

但是，直接把抓来的网页喂给模型是灾难性的：原始网页里有 HTML 标签、导航菜单、广告、乱码、无数转载重复，还有骂战和隐私信息。模型读什么学什么——**垃圾进，垃圾出**。

所以每一篇网页在入库前，都要走完一条残酷的流水线。下面这条 5 道工序的迷你流水线，处理的是 6 篇模拟「原始网页」——亲手把它们送进去：

一步步点「下一道工序 →」，重点观察三件事：

1. **「英文体育新闻」在语言识别这步消失了**——注意淘汰原因写的是：它不是被扔掉，而是转去英文流水线。真实的多语言模型会为每种语言开一条线。
2. **「转载文章」死在去重这一步**。它和「新闻报道」内容几乎一样——重复数据是双重浪费：白白多算一遍，还会让模型把这篇文章「背下来」而不是学规律（第 2 章讲过的过拟合，在数据侧同样成立）。
3. **两篇幸存文档也都动过手术**：新闻报道里的「点击下载APP领红包！」在质量过滤时被删掉，学习经验帖里的手机号被打码成 138\*\*\*\*5678。清洗不只是「扔文档」，更多的是「治病救文」。

走到终点你会看到：6 篇原始网页只剩 2 篇入库，存活字符不到一半。真实规模下更残酷——以公开的大型网页语料项目的报告为参照（2024-2025 年），原始抓取数据里**最终只有百分之几**能进训练集。

:::caution 这条流水线在哪里被简化了？
真实的清洗流水线有**几十道**工序，这里只演示了最有代表性的 5 道；很多过滤器（质量打分、毒性识别）本身就是训练出来的小模型，而不是简单规则——用模型造模型的数据，是行业常态。另外「只剩百分之几」是公开数据集报告的量级，各家标准松紧差异很大，不是精确数字。
:::

**知名开源预训练数据集的演进**（规模按发布时的报告，仅供量级参考）：

| 数据集 | 年份 | 规模 | 一句话特点 |
| --- | --- | --- | --- |
| C4 | 2019 | 约 156B token | Common Crawl + 规则清洗的开山之作，规则简单粗暴 |
| The Pile | 2020 | 约 300B token | 22 个来源人工调配，首个强调「配比」的公开数据集 |
| RefinedWeb | 2023 | 约 600B token 公开 | 证明「只用网页、清洗到位」就能媲美精调配比 |
| FineWeb | 2024 | 约 15T token | 96 次 Common Crawl 快照淘洗而成，清洗全流程开源可复现 |

**去重的两个层次。** 精确去重（整篇哈希对比）只能抓一模一样的副本；真正的难题是「几乎重复」——转载时改了个标题、加了行「【转载】」。主流方案是 **MinHash** 一类的模糊去重：把每篇文档压缩成一个小「指纹」，指纹相近就判为重复。直觉：不逐字对比全文，而是抽查若干「特征片段」是否大量重合。

**质量分类器。** 「高质量」没法用规则定义，所以业界的做法是：拿公认的高质量文本（如百科、教科书）当正样本，训练一个小分类器，再让它给全网每篇文档打质量分，低于阈值的淘汰。FineWeb-Edu（2024）更进一步：专门训了「教育价值」打分器，筛出的子集显著提升了模型的知识类能力。

**数据用几遍？** 语料不够时能不能把同一批数据训多个 epoch？经验结论（2023 年前后的多篇研究）：重复 2~4 遍代价尚可接受，再多则收益迅速衰减、过拟合风险上升——这也是各家拼命扩充**新**数据的原因。

**数据污染伏笔。** 清洗还有一道容易被忽视的工序：把公开评测集的题目从训练数据里剔除。做不干净的话，模型等于「提前背了考题」，评测分数全面虚高——第 7 章讲评测时会专门算这笔账。

</DeepDive>

## 小结

:::tip 本节要点
- 十几万亿 token 的语料只能来自互联网，网页抓取占大头，代码、书籍、论文各补一种「营养」。
- 原始网页必须经过正文提取、语言识别、质量过滤、去重、隐私毒性处理等几十道工序，最终存活率只有百分之几。
- 重复数据 = 白算 + 背题；清洗质量和数据配比直接决定模型上限——「垃圾进，垃圾出」。
:::
