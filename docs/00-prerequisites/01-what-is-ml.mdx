# 0.1 什么是机器学习

## 先猜一个分数

期末考试前，班主任想预测每个同学的成绩。她手里有全班的历史数据：每人**每天学习几小时**，以及**上次考了多少分**。

凭经验，她心里有杆秤：*「学得越久，分数越高，大概每多学一小时能多拿几分。」*

把这杆秤写成式子，就是一条直线：

$$
\text{分数} \approx w \times \text{学习时间} + b
$$

这里 $w$ 和 $b$ 是两个待定的数字：$w$ 是「每多学一小时值多少分」，$b$ 是「完全不学大概能考多少」。**这条带着两个可调数字的直线，就是一个最简单的「模型」**——数据进去，预测出来。

问题来了：$w$ 和 $b$ 到底取多少，这条线才「猜得最准」？

## 你刚才做的事，就是机器学习

回顾一下你刚才的操作流程：

1. 看着直线和数据点的**偏差**（红色虚线）；
2. 觉得偏差太大，就**调整** $w$ 和 $b$；
3. 反复调，直到偏差**尽可能小**。

再点「🤖 让机器自动学」，机器做的是完全同一件事，只是不用人眼看、不用手拖——它用算法自动完成「衡量偏差 → 调整参数」的循环。

这就是机器学习的完整定义：

:::info 一句话定义
**机器学习（<Term id="machine-learning">Machine Learning</Term>）= 不靠人手写规则，而是让程序根据数据自动调整<Term id="parameter">参数</Term>，使模型的预测偏差尽可能小。**
:::

三个词值得记住，后面每一章都会用到：

- **<Term id="model">模型</Term>**：带可调参数的函数（这里是一条直线）。
- **<Term id="parameter">参数</Term>**：那些可调的数字旋钮（这里是 $w$ 和 $b$，一共 2 个）。
- **训练**：自动调参数、让偏差变小的过程（你点的那个按钮）。

## 从 2 个参数到 2000 亿个

你可能会想：一条直线也太简单了，跟 ChatGPT 有什么关系？

关系是——**原理一模一样，只是规模不同**：

| | 刚才的直线 | 大语言模型（2025 年的旗舰级） |
| --- | --- | --- |
| 输入 | 学习时间（1 个数字） | 上文（几万个<Term id="token">词元</Term>） |
| 输出 | 预测分数（1 个数字） | 下一个词的概率（几万个数字） |
| 参数量 | 2 个 | 数百亿 ~ 数万亿个 |
| 训练数据 | 12 个点 | 十几万亿词的文本 |
| 训练方式 | 调 $w, b$ 减小偏差 | 调所有参数减小偏差——完全同款 |

后面整本书要讲的，无非是三件事：

1. 模型的「函数」长什么样？（第 2、3 章：神经网络、Transformer）
2. 「偏差」怎么定义？（第 1、2 章：损失函数）
3. 参数那么多，怎么自动调？（第 0.4、2 章：梯度下降、反向传播）

:::caution 这个类比在哪里不准确？
「班主任的直线」是**有监督回归**：每条数据都带着标准答案（真实分数）。大模型的预训练其实不用人工标答案——它拿一句话的前半段当输入、后半段的下一个词当「答案」，答案就藏在文本自己身上（第 4 章细讲）。另外，直线只能表达「线性」规律，大模型用的函数要弯曲、复杂得多——这正是第 2 章神经网络要解决的问题。
:::

<DeepDive title="形式化定义与最小二乘解">

**监督学习的标准表述。** 给定数据集 $\mathcal{D} = \{(x_i, y_i)\}_{i=1}^{n}$，我们选定一个带参数 $\theta$ 的函数族 $f_\theta$（假设空间），并定义<Term id="loss">损失函数</Term>来量化预测偏差。回归问题最常用均方误差（MSE）：

$$
L(\theta) = \frac{1}{n} \sum_{i=1}^{n} \bigl(f_\theta(x_i) - y_i\bigr)^2
$$

训练即求解优化问题 $\hat{\theta} = \arg\min_\theta L(\theta)$，这个框架叫**经验风险最小化**（Empirical Risk Minimization）。

**线性模型有闭式解。** 对 $f_{w,b}(x) = wx + b$，令 $\partial L / \partial w = 0$ 和 $\partial L / \partial b = 0$，解得：

$$
\hat{w} = \frac{\sum_i (x_i - \bar{x})(y_i - \bar{y})}{\sum_i (x_i - \bar{x})^2}, \qquad \hat{b} = \bar{y} - \hat{w}\,\bar{x}
$$

其中 $\bar{x}, \bar{y}$ 是均值。这就是**最小二乘法**——上面实验里「让机器自动学」按钮最终收敛到的位置就是它。

用 NumPy 验证只需几行：

```python
import numpy as np

x = np.array([0.5, 1, 2, 2.5, 3.5, 4, 5, 6, 7, 7.5, 8.5, 9.5])
y = np.array([29, 33, 35, 44, 42, 52, 57, 58, 70, 64, 78, 80])

w = ((x - x.mean()) * (y - y.mean())).sum() / ((x - x.mean())**2).sum()
b = y.mean() - w * x.mean()
print(f"分数 ≈ {w:.2f} × 时间 + {b:.2f}")   # ≈ 5.72 × 时间 + 26.34
```

**为什么大模型不用闭式解？** 闭式解只对线性模型 + 平方损失这类特殊组合存在。神经网络是高度非线性的，$L(\theta)$ 没有解析解，只能用迭代法（梯度下降，见 0.4 节）一步步逼近。此外参数量达数千亿时，即使存在闭式解，求解所需的矩阵运算也远超算力可行范围。

</DeepDive>

## 小结

:::tip 本节要点
- **模型**是带可调参数的函数；**训练**是自动调参数让预测偏差变小的过程。
- 偏差需要一个明确的量化标准（本例中是均方误差），机器才知道「调得好不好」。
- 大模型与这条直线原理相同，差别只在参数量（2 个 → 数千亿个）和函数复杂度。
:::
