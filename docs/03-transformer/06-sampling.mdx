# 3.6 采样与生成：三个旋钮决定它怎么说话

## 流水线尽头，还差最后一步

3.5 节的流水线走到尽头，模型交出来的是一张**概率表**，不是句子。从概率表到落在屏幕上的那个字，中间还有一道工序：**怎么选**。

最老实的选法是每次都拿概率第一名（贪心）。听起来稳妥，实际写长文很容易翻来覆去说车轱辘话——高概率的路径未必像人话（原因见深入层）。所以主流做法是**抽签**，但抽签前允许你动三个旋钮改造骰子：<Term id="temperature">温度</Term>、<Term id="top-k">top-k</Term>、<Term id="top-p">top-p</Term>。

亲手转一转：

<SamplingPlayground />

灰色空心条是模型给的**原始概率**，蓝色实心条是三道工序处理后**真正参与抽签**的概率。请按顺序做四个实验：

**实验 ①：温度拉到 0.1。** 分布被削成一根独苗，几乎每次都选第一名——组件会提示：T 趋近 0 就是「贪心解码」。适合要答案稳定的场景（数学、写代码）。

**实验 ②：温度拉到 2.5，点「✍️ 自动写 15 字」。** 分布被抹平，冷门字频频上位，几句话就语无伦次。高温 = 创造力和胡说八道一起来。

**实验 ③：温度调回 1.0，top-k 选「前 3 名」。** 第 4 名之后全部 ✂️ 出局，剩下 3 个候选把概率蛋糕重新分掉。

**实验 ④：top-p 拉到 0.7。** 按概率从高到低累加，刚凑够 70% 就收手，圈外全砍。注意它和 top-k 的区别：**核心圈的大小会自动伸缩**——分布很尖时圈里可能只剩 1 个字，分布平缓时能容七八个。

三道工序的固定顺序：**温度（重塑分布）→ top-k（留前 k 名）→ top-p（留核心圈）→ 重新归一化 → 掷骰子**。

## 骰子改造工艺，至此讲全

回头看，这条线我们埋了很久：[0.3 节](../00-prerequisites/03-probability.mdx)说「生成文字 = 掷特制骰子」；[1.1 节](../01-language-models/01-what-is-lm.mdx)你按过「🎲 按概率抽一个字」——当时模型给什么骰子就掷什么。本节补上了最后一环：掷之前还能**改造骰子**。你在各家 API 里见到的 `temperature`、`top_p` 参数，就是这里的旋钮，一模一样。

:::caution 这个实验在哪里不准确？
组件里的概率来自第 1 章那个查表迷你模型，不是真正的 Transformer。但结论不打折扣：**采样发生在模型之后**，它面对的只是「一张概率表」，不管这张表是查表查出来的还是千亿参数算出来的，改造工艺完全一致。另外，真实产品往往还有 frequency penalty、repetition penalty（给已出现过的词降分、抑制复读）等旋钮，思路同源，这里不展开。
:::

<DeepDive title="三种采样的精确定义">

**温度**作用在 logits 上：

$$
p_i = \mathrm{softmax}(z / T)_i = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}
$$

$T \lt 1$ 拉大分差（分布变尖），$T \gt 1$ 抹平分差（分布变平）。组件里是对已归一化的概率取 $1/T$ 次幂再归一化——两者等价，因为 $p_i^{1/T} \propto (e^{z_i})^{1/T} = e^{z_i/T}$。

**$T \to 0^+$ 的极限**：最大的 logit 独占全部概率，采样退化为 $\arg\max$，即贪心解码——这就是实验 ① 看到的现象。

**top-k**：候选集固定为概率最大的 $k$ 个，集合内重新归一化，其余概率归零。

**top-p（核采样）**：将候选按概率降序排列，取累计概率恰好达到 $p$ 的**最小前缀集合**，在集合内归一化。出处是 Holtzman et al.（2019）——该论文同时给出了「高概率路径会退化成复读机」的系统证据：贪心和束搜索生成的文本重复率远高于人类文本，这正是开放式生成不用束搜索（beam search，一种始终保留 B 条最优路径的搜索法，翻译等有标准答案的任务常用）的原因。

**典型默认值**（2025–2026 年常见）：$T \approx 0.7 \sim 1.0$，top-p $\approx 0.9$，top-k 关闭或取 40～50；代码、数学等要求确定性的场景常把 $T$ 压到 0.2 以下甚至用贪心。

</DeepDive>

## 小结

:::tip 本节要点
- 模型给概率，采样定文字；这一步不涉及训练，是纯「推理期工艺」。
- 温度重塑分布：低温保守稳定，高温奔放混乱；T → 0 即贪心。
- top-k 留前 k 名，top-p 留「累计刚够 p」的自伸缩核心圈，都是砍长尾防怪词。
- 大模型每次回答不一样 = 采样的随机性；想要稳定，调低温度。
:::
